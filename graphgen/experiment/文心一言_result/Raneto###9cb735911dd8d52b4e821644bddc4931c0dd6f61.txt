已知Dockerfile中的最小单元是以指令开头的行，并且从0开始给每个指令行进行编号，即第n个指令行对应n-1。若(0,1)表示第二个指令行依赖于第一个指令行，注意前后逻辑关系，现在需要得到Dockerfile的所有指令行之间的依赖关系列表。要求返回一个列表，其中每个元素的形式类似于(0,1)这种形式。


FROM node:18.17.0-alpine
EXPOSE 3000
ENV HOST 0.0.0.0
ENV PORT 3000
WORKDIR /opt/raneto
COPY . /opt/raneto
RUN npm install --omit=dev
CMD ["npm", "start"]

请给出Dockerfile的所有指令行之间的依赖关系列表，只给出答案和一条注释描述即可，每个依赖关系后面跟着注释描述，不需要其余描述。

[(1, 0),  # EXPOSE 依赖于 FROM，因为暴露端口是基于基础镜像的  
 (2, 1),  # 第一个 ENV 依赖于 EXPOSE，因为环境变量的设置可能依赖于端口暴露  
 (3, 2),  # 第二个 ENV 依赖于第一个 ENV，因为环境变量可能相互依赖  
 (4, 3),  # WORKDIR 依赖于第二个 ENV，因为设置工作目录可能依赖于环境变量  
 (5, 4),  # COPY 依赖于 WORKDIR，因为文件复制的目标目录需要提前设置  
 (6, 5),  # RUN 依赖于 COPY，因为安装依赖需要在文件复制完成后进行  
 (7, 6)]  # CMD 依赖于 RUN，因为启动命令需要在所有依赖安装完毕后执行
