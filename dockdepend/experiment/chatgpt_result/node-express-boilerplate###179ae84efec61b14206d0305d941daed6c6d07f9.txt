已知Dockerfile中的最小单元是以指令开头的行，并且从0开始给每个指令行进行编号，即第n个指令行对应n-1。若(0,1)表示第二个指令行依赖于第一个指令行，注意前后逻辑关系，现在需要得到Dockerfile的所有指令行之间的依赖关系列表。要求返回一个列表，其中每个元素的形式类似于(0,1)这种形式。


0 FROM node:alpine
1 RUN mkdir -p /usr/src/node-app && chown -R node:node /usr/src/node-app
2 WORKDIR /usr/src/node-app
3 COPY package.json yarn.lock ./
4 USER node
5 RUN yarn install --pure-lockfile
6 COPY --chown=node:node . .
7 EXPOSE 3000

请给出Dockerfile的所有指令行之间的依赖关系列表，只给出答案和一条注释描述即可，每个依赖关系后面跟着注释描述，不需要其余描述。
注意不是只判断前后两条指令行间的依赖关系，存在(1,3)这种情况，且前后两条指令不一定有依赖关系，依赖关系定义是即不可交换的偏序关系，交换后会导致原镜像构建失败。

[
    (0, 1),  # 基础镜像需要在创建目录之前定义
    (1, 2),  # 需要在设置工作目录之前创建并赋予权限
    (2, 3),  # 设置工作目录后才能复制文件
    (3, 4),  # 需要在切换用户之前复制依赖文件
    (4, 5),  # 切换到node用户后才能安装依赖
    (2, 6),  # 设置工作目录后才能复制源代码并赋予权限
    (5, 6),  # 安装依赖后才能复制应用代码
]
